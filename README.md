# GEN-Z VPN — Web Interface & Billing System

Современный, быстрый и безопасный веб-интерфейс для управления личным VPN-сервисом с полноценной системой авторизации, динамическим биллингом (кошельком) и автоматическим управлением тарифными планами на базе СУБД PostgreSQL.

---

## 🚀 Актуальность и Назначение
Проект представляет собой full-stack веб-приложение, спроектированное для автоматизации бизнес-процессов подписок на защищенные сетевые протоколы. Сервис эффективно решает три ключевые задачи:
1. **Безопасность сессий:** Защита пользовательского трафика и управление доступом к конфигурациям WireGuard/OpenVPN.
2. **Автоматизация биллинга:** Списание средств за тарифные планы в реальном времени внутри транзакций СУБД.
3. **Синхронизация состояний:** Мгновенное обновление UI (React) при изменениях в БД без принудительной перезагрузки страниц.

## 💾 Архитектура Базы Данных (PostgreSQL)

Система построена на реляционной схеме со строгой каскадной связью `One-to-One` / `One-to-Many` между таблицами пользователей и их активных подписок.

### 1. Таблица `users`
Хранит базовые учетные данные пользователей и текущий финансовый баланс кошелька.
* `id` (SERIAL, PRIMARY KEY): Уникальный идентификатор.
* `email` (VARCHAR(100), UNIQUE): Электронная почта пользователя (логин).
* `password` (VARCHAR(255)): Хэшированный пароль.
* `balance` (INT, DEFAULT 0): Текущий баланс счета в рублях (защищен от строковых типов на уровне API).

### 2. Таблица `subscriptions`
Связана с таблицей пользователей внешним ключом и отвечает за статус тарифа.
* `id` (SERIAL, PRIMARY KEY): Идентификатор записи.
* `user_id` (INT, REFERENCES users(id) ON DELETE CASCADE): Связь с аккаунтом.
* `plan_name` (VARCHAR(50), DEFAULT 'Пробный'): Текущий активный тарифный план (`Пробный`, `Месячный`, `Годовой`).
* `created_at` (TIMESTAMP): Дата оформления подписки.

## 🌐 Спецификация REST API (Серверный слой)

Бэкенд функционирует на порту `5000` и предоставляет следующие эндпоинты, обменивающиеся данными строго в формате `JSON`:

* `POST /register` — Регистрация нового аккаунта. При успешном создании автоматически генерирует запись в таблице `subscriptions` со статусом `Пробный`.
* `POST /login` — Аутентификация пользователя и выдача email для локальной сессии.
* `GET /get-profile?email=...` — Получение актуального баланса и названия тарифа напрямую из PostgreSQL. Защищен от падения: если запись подписки отсутствует, возвращает дефолтный статус.
* `POST /update-balance` — Универсальный шлюз транзакций. 
  * Принимает `{ email, amount, planName }`.
  * При `amount > 0` производит пополнение баланса.
  * При `amount < 0` и передаче `planName` проверяет баланс, списывает стоимость тарифа и выполняет операцию атомарного обновления/создания подписки (`INSERT ... ON CONFLICT` / подзапросы с гарантированным ограничением `LIMIT 1`).

## ⚙️ Команды Управления и Автоматизации (Makefile)

Для минимизации рутинных терминальных команд развертывание инкапсулировано в `Makefile`:

* `make install` — Потоковая установка зависимостей фронтенда и бэкенда.
* `make dev` — Запуск локального сервера разработки Vite для фронтенда.
* `make stop` — Экстренное уничтожение зависших процессов Node.js, занимающих порты.
* `make build` — Компиляция оптимизированного production-билда интерфейса.
